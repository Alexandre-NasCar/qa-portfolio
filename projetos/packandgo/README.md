# PackAndGo — Projeto de Quality Assurance

## Visão geral

O PackAndGo é um projeto educacional e autoral voltado ao planejamento e à descoberta de viagens.

O projeto é utilizado como ambiente prático para aplicar conceitos de desenvolvimento e Quality Assurance desde as etapas iniciais, conectando requisitos, funcionalidades, riscos, testes, evidências e aprendizados.

---

## Objetivo do projeto

Desenvolver uma aplicação Web que permita ao usuário explorar destinos, planejar viagens, registrar experiências e organizar sua jornada de forma simples e visual.

No contexto de QA, o objetivo é estruturar uma estratégia de qualidade que acompanhe a evolução do produto e valide seus principais fluxos.

---

## Estratégia de testes

A estratégia de testes será estruturada por risco e prioridade.

A trilha utilizada será:

> Requisito → Risco → Cenário → Caso de teste → Execução → Evidência → Resultado → Aprendizado

Os testes serão organizados inicialmente em:

- testes manuais funcionais;
- testes de interface;
- testes de responsividade;
- testes de navegação;
- testes de regras de negócio;
- testes de autenticação;
- automação Web;
- testes de regressão.

---

---

## Riscos iniciais identificados

- falha na persistência da sessão;
- duplicidade de e-mail ou CPF;
- navegação para seções incorretas;
- perda de dados armazenados no navegador;
- comportamento inconsistente em telas menores;
- botões sem ação;
- estados visuais pouco claros;
- ausência de mensagens de erro;
- inconsistência entre requisitos e implementação.
